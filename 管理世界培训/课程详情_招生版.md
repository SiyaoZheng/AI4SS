## M1. 范式转变 + Claude Code 入门

AI 正在改变学术研究的工作方式，但许多研究者对此感到困惑：

- 市面上 AI 工具层出不穷，到底该用哪一个？
- 跟 ChatGPT 聊天和真正用 AI 做研究，差距在哪里？
- 担心 AI 生成的代码或结果不可靠，不知道如何验证？

本模块帮助你建立清晰的认知框架，理解当前 AI 能力的实质边界，以及 Claude Code 作为研究智能体与普通 AI 对话框的本质区别：它不是"你问一句、它答一句"的聊天机器人，而是一个能自主读取数据、规划步骤、执行代码、发现错误并自我修复的 AI 研究助手。

### 专题要点

- AI 能力的三次跃迁：从"会写文字"到"会推理"再到"会自主执行任务"，三个阶段对社会科学研究各自意味着什么
- Claude Code 与 ChatGPT、Copilot 的本质区别：它是一个研究智能体——主动读取你的数据文件，自主运行分析脚本，遇到报错自己诊断修复，管理整个研究项目，而不只是被动地回答你的问题
- 研究者角色转变：从"自己写代码"到"描述需求、AI 执行"，核心技能从掌握语法转向清晰表达研究意图
- 如何验证 AI 的输出：什么可以信任，什么必须手工核查——这是贯穿工作坊的核心原则
- 主流 AI 编程智能体对比：Claude Code、Trae、Codex、OpenClaw——为什么选择 Claude Code 作为研究工具
  - **Claude Code**（本工作坊核心工具）：Anthropic 官方智能体，深度集成 Claude 模型的工具调用能力，可读写本地文件、自主执行代码、自动调试纠错，对 Stata/R/Python 等研究语言支持最好，是目前实证研究者最成熟的本地编程智能体
  - **Trae**：字节跳动推出的 AI 编程工具，界面友好、上手快，但在自主执行、多步骤调试和复杂项目管理方面能力有限，更适合前端/应用开发场景
  - **Codex**：OpenAI 编程智能体，代码理解和生成能力强，但生态封闭、模型能力对研究任务支持不如 Claude Code 灵活
  - **OpenClaw**：基于 Claude Code 扩展的自主研究 Agent，可在远程服务器持续运行、自主协调多步骤研究流程，是 Claude Code 的"无人值守版"

### 现场演示

讲师用一句话提示词生成完整 OLS 回归工具（含文件上传、描述统计、变量选择、回归输出、可视化）。

## M2. Claude Code 核心操作演示

很多人对 Claude Code 的印象还停留在"会聊天的代码补全工具"。事实上，Claude Code 是一个能在你的项目目录里真正工作的 AI 助手：它主动读取数据文件、生成并运行脚本、看到错误后自己修复、再告诉你结果。

本模块以讲师演示为主，目标是让每位学员直观理解 Claude Code 的完整工作流程，看清 AI 如何自主完成研究任务。

### 专题要点

- 装好就能用：安装、账号登录、第一次启动，全程一行命令
- 你能看到 AI 在做什么：Claude Code 执行每一步操作时都会显示，你可以随时批准或叫停，不是黑箱
- 基本操作：让 AI 读取你的数据文件、生成分析脚本、运行并返回结果
- 告诉 AI 你的研究背景：如何用一个配置文件让 Claude Code 记住你的数据路径、变量定义和分析惯例，下次启动无需重复说明
- 建立核查节奏：什么时候该相信 AI 的输出，什么时候要亲自检查固定效应、样本量、变量编码

### 教师演示

讲师在示例研究项目目录下，用自然语言完成三个递进任务：查看数据结构 → 生成描述统计 → 绘制分布图，学员全程观察 Claude Code 的操作过程。

## M3. Claude Code 进阶：项目工作流

M1 建立了认知，M2 直观展示了操作流程。但当你开始把 Claude Code 用于真实研究项目时，会遇到新的问题：

- 上次对话告诉过 AI 的数据路径和变量定义，这次启动又要重新说一遍
- 项目文件越来越多，AI 不知道哪个 do file 对应哪个分析步骤
- 代码报错了，还是需要自己看错误信息，手动修改，重新运行
- 不知道 Claude Code、Claude.ai、Cursor 三者之间该如何选择和搭配

本模块紧接 M2，在你已经看过 Claude Code 操作演示的基础上，介绍如何把它集成到研究项目的日常工作流中，让 AI 在有记忆、有结构的环境里持续工作。

### 专题要点

- 项目记忆配置：把数据路径、文件结构、分析惯例写在一个配置文件里，每次启动 Claude Code 自动读取，无需重复说明
- 多文件项目管理：do file / R script 的自动生成、组织与追踪，AI 知道你的项目在哪个阶段
- 让 AI 自主调试：代码报错后，Claude Code 自动读取错误、修改、重跑，直到跑通为止，无需手动介入
- 灵活的协作模式：Claude Code 生成代码，研究者在自己熟悉的 Stata/R 环境中执行并返回结果
- 工具选择逻辑：什么场景用 Claude Code、什么场景用 Claude.ai 网页版、什么场景用 Cursor

### 教师演示

讲师在本地项目中完成配置，然后用自然语言下达一个完整分析任务——从数据导入到回归输出——学员观察 Claude Code 全程自主执行，包括处理中途的报错。

## M4. 智能体数据工程：从原始数据到分析样本

在实证研究中，数据清洗往往占据整个项目一半以上的时间，却很少出现在论文里。你可能有这样的经历：

- 多个数据库下载的变量命名不一致，手动对照合并费时费力
- 宽格式/长格式转换、winsorize、缺失值处理，每次写代码都要重新查语法
- 年报、政府报告里的关键数字分散在非结构化文本中，手工提取根本不现实
- CSMAR 数据合并专利库，匹配率核查没有统一标准

本模块演示如何用自然语言向 Claude Code 描述数据操作需求，由这个研究智能体自主选择工具和方法，生成并执行 Stata/R/Python 代码，同时保留完整操作记录以便复现。

### 专题要点

- 自然语言数据操作：宽转长、变量重命名、跨库合并、模糊匹配、winsorize——描述需求，AI 生成可执行脚本；你只需核查结果，不需要写语法
- 非结构化数据批量结构化：从年报文本、政府报告、企业公告中提取结构化信息，不再逐条手工录入
- 中国特色数据源：CSMAR/Wind、统计年鉴、工商数据、社交媒体数据的 AI 辅助处理方案
- 数据质量核查：让 Claude Code 生成缺失值报告、匹配率审计，追溯原始记录验证，建立可复现的数据管道

### 教师演示

讲师演示 AI 辅助跨库数据合并：完成匹配、不匹配项处理与匹配率核查。

## M5. AI 赋能计量分析：描述研究设计，AI 跑回归

实证研究者最头疼的不是"不懂方法"，而是"把方法翻译成代码"——查语法、调格式、排错，耗费的时间远超思考研究设计本身。更隐蔽的问题是：AI 生成的代码跑通了、结果出来了，但你真的能信任它吗？固定效应加对了吗？样本量为什么少了 3000？变量编码有没有被 AI 误解？

本模块的核心不是教计量方法，而是教你两件事：**如何把研究设计准确描述给 AI**，以及 **AI 跑出结果后应该核查什么**。无论你用的是 OLS、DiD、IV、RD 还是合成控制，原理相同。

### 专题要点

- 结构化描述研究设计：把观测单位、因变量、核心解释变量、固定效应、标准误聚类层级说清楚——这五个要素讲明白了，Claude Code 就能生成完整的基准回归代码，方法不限
- AI 执行，你核查：Claude Code 跑出结果后的核查清单——固定效应是否正确加入、样本量是否与预期一致、变量编码是否被误解、标准误是否在合理范围——建立"信任但验证"的习惯
- 从一张表到全套检验：基准回归跑通后，一句话让 AI 批量生成平行趋势检验、安慰剂检验、稳健性检验变体，自动汇总为对比表
- 结果可视化：描述想要的图（事件研究图、系数对比图、边际效应图、地图），AI 生成绘图代码并迭代修改——配色、标注、分辨率一步到位，达到期刊投稿标准

### 教师演示

- 讲师用自己的研究（或示例数据），用自然语言描述一个回归设定，Claude Code 生成代码并运行，演示按核查清单逐项检查输出
- 讲师描述一个 DID 设定，Claude Code 生成事件研究图
- 将回归结果转化为论文级图表——调整配色、标注、布局以满足期刊投稿要求

## M6. 学术成果展示：从图表到 Slides

实证研究者经常面临"结果跑出来了，但展示效果跟不上"的困局：

- 回归结果有了，但画出的图总是跟 Top Journal 里看到的差距很远——配色、标注、分辨率，每个细节都要手工调很久
- 学术会议、开题答辩、课题组汇报——每次都要花大量时间做 Slides，内容多、逻辑乱、不好看
- 同一个结果要适配不同场景（论文投稿用 PDF 图、会议用宽屏 Slides、申报书用 Word 表格），格式要求各不相同

本模块演示如何用 Claude Code 将分析结果一键转化为论文级图表和学术汇报 Slides，把"做展示"的时间从几小时压缩到几分钟。

### 专题要点

- 论文级可视化迭代：给 Claude Code 看一篇 Target Journal 的典型图表 → 描述你的数据和想要的图 → AI 生成初版 → 你提修改意见（"置信区间太粗了"、"横轴标签改成中文"）→ AI 自动迭代，直到满意
- 从数据到 Slides：把回归结果、图表、文字结论整合为学术汇报 Slides——Claude Code 直接输出 reveal.js HTML 或 PPT 文件，包含完整的标题、图表、要点和过渡逻辑
- 多场景适配：同一套分析结果，快速切换为不同展示格式——论文投稿的多面板组合图、会议 Slides 的宽屏布局、申报书里的表格排版
- 风格一致性：定义一次视觉风格（配色、字体、布局），后续所有输出自动统一，摆脱手动调格式

### 教师演示

讲师用 M5 中跑出的回归结果，让 Claude Code 生成一份学术会议级 Slides（含标题、核心图表、要点结论），学员观摩全过程。

## M7. Research Agent：从 AI 赋能到自主研究流程

前面每个模块解决的是研究流程中的单个环节——数据清洗、回归分析、成果展示。这一模块讨论的是更进一步的问题：让 Claude Code 把这些环节串联起来，自主协调完成从文献检索到结果汇报的完整链条。

目前已有社会科学研究者将这套流程封装为开源工具，可以直接使用：

- Clo-Author（Hugo Sant'Anna, 香港大学）：覆盖从文献综述到期刊投稿全流程，研究者用自然语言下达任务，AI 调度多个专用助手分工完成
- claude-code-my-workflow（Pedro Sant'Anna, Emory 大学应用计量课程）：面向实证研究者的 Claude Code 配置模板，可直接 fork 使用

### 专题要点

- Research Agent 的工作原理：AI 如何拆解研究任务、调用外部工具、完成子任务、再汇总结果
- Claude Code 作为研究基础设施：连接文献数据库、数据 API、本地代码执行，让 AI 跨越多个工具完成任务
- Clo-Author 实操解析：这套框架能做什么、不能做什么，在自己的研究项目里如何上手
- 门槛分层：从 claude.ai 浏览器端到本地命令行，不同使用深度对应的能力边界和学习成本

### 现场演示

以讲师在研项目为案例，演示 Claude Code 驱动的完整 Agent 工作流：文献检索 → 数据清洗 → 回归分析 → 结果汇报，全程展示 AI 的推理过程和操作记录。

## M8. 综合实战 + 圆桌

圆桌讨论，围绕三个问题：

1. Claude Code 哪些环节真正有价值？哪些输出需要高度警惕、必须手工核查？
2. 在各自的研究领域，Claude Code 能扮演什么角色？
3. 中国研究情境下的适用边界：数据保密、模型访问、结果可复现性等实际问题
