# AI4SS 工作坊：AI Agent时代的社会科学研究

**课程类型**：集中工作坊（一天半）
**授课者**：郑思尧（上海交通大学）
**面向受众**：政治学、公共管理、社会学方向研究生及青年学者
**先修要求**：无编程基础要求；具备基础定量研究方法知识（了解回归分析）

---

## 核心框架

生成式AI已成为社会科学研究的**基础设施**——研究者已经在其之上开展工作，而非仍在观望是否要用。

但这一基础设施的设计逻辑源自软件工程，与社科研究的方法论要求之间存在**结构性错配**：

| AI基础设施的逻辑 | 社科研究的要求 |
|---|---|
| 以生成流畅内容为目标 | 因果识别需要研究者判断 |
| 以用户满意度优化 | 测量效度需要理论论证 |
| 以单次任务为单位 | 研究是跨月乃至跨年的持续过程 |
| 黑箱优化 | 分析决策需可追溯、可解释 |

本工作坊帮助研究者**识别并弥合**这一错配，培养两种核心能力：
1. **判断力**：识别AI默认行为对研究逻辑的偏离
2. **改造能力**：重塑AI行为使之符合研究规范

## M1. 导论：AI基础设施与社科研究的结构性错配

**时长**：90分钟
**性质**：讲授为主，课堂讨论

我们正在经历一场关于知识生产的底层革新。大语言模型不只是一个写作工具，它正在成为研究者的**协作者、模拟者与对话者**——以极快速度、极高语义一致性给出看似"合理"的输出。但这种合理性，是否真的是理解？

本模块帮助你建立使用AI的**底层判断力**，而非使用手册。

### 专题要点

- **LLM工作原理（直觉化）**：Transformer架构与注意力机制的研究者视角解读；为什么它能"懂上下文"，又为什么会"一本正经地胡说八道"；Token预测与统计推断的本质差异
- **主流模型比较**：GPT-4o、Claude 4、Gemini、DeepSeek、文心一言的能力边界与适用场景；何时用哪个模型
- **强项与局限**：零样本学习与小样本能力；上下文窗口限制；内在偏见与幻觉的系统性来源
- **结构性错配的五个维度**：流畅性≠准确性；满意度优化≠方法论严谨；快速产出≠可复现性；通用补全≠理论建构；单次任务≠研究全生命周期
- **批判性使用的实践原则**：如何与AI"协作而不盲信"；分析决策的可追溯性要求

### 工具演示

- **OpenAI Playground / Claude.ai**：现场演示同一研究问题的不同提示词输出差异
- **幻觉测试**：要求模型引用一篇具体文献，实时验证其准确性
- **可视化动画**（选用）：Transformer自注意力机制直觉解释

### 课堂练习

- **幻觉体验**：学员要求模型解释一个社科术语（如"沉默的螺旋"），然后当场在Google Scholar验证模型给出的"文献来源"是否真实存在

## M2. 研究设计、假设生成与文献调研

**时长**：105分钟
**性质**：讲授 + 上机实践

研究设计的难点不在于技术门槛，而在于它要求研究者**同时具备理论判断力、方法意识与对领域文献的深度感知**。本模块正是为"研究起点阶段"的困难而设计：面对大量文献不知从何入手，有想法却难以发展为可检验的假设，或无法系统地开展文献综述。

### 专题要点

- **研究问题的AI辅助发展**：如何利用AI进行头脑风暴式的研究问题发散；LLM的"对话场"如何帮助识别研究空白；边界与陷阱——AI建议的研究方向为何往往缺乏理论深度
- **文献识别与脉络分析**：AI驱动的语义搜索（超越关键词匹配）；构建领域知识图谱的实践方法；从单篇文献到演化脉络
- **假设生成与推演**：使用LLM围绕研究问题构建假设；AI自动推演假设的合理性与局限；研究者如何对AI假设进行批判性评估
- **引用文献的"meta信息"分析**：理解文献在不同研究中的角色与语境变化，从而提升引用的方法论适配度
- **实践中的注意事项**：AI摘要后必须回原文核对的关键节点；防止引用错误；版权与学术诚信

### 使用工具

- **文献语义搜索**：Elicit、Semantic Scholar、Perplexity（学术模式）
- **文献对话**：ChatPDF、Claude（上传PDF直接问答）
- **文献知识图谱**：Research Rabbit、Connected Papers
- **大型语言模型**：ChatGPT / Claude 用于头脑风暴与假设推演

### 上机练习

- **练习：研究方案头脑风暴**：学员以自己领域的一个初步想法，使用ChatGPT进行假设发展，然后小组讨论：AI给出的假设哪些有理论价值，哪些缺乏可操作性
- **案例：快速综述起草**：提供一个研究主题（如"社交媒体与政治参与"），学员使用Elicit检索5篇文献，提取主要发现点，并用AI生成初版综述段落，再与同伴互评其方法论准确性

## M3. AI辅助数据收集与预处理

**时长**：105分钟
**性质**：讲授 + 工具演示

在社会科学研究中，"找数据"和"清洗数据"几乎占据了整个研究流程的一半时间。本模块聚焦**AI驱动的数据收集与预处理流程**，从原始信息提取到结构化建表的核心技能。

**这门课不教你编程**，但教你如何用AI工具完成传统上需要编程才能完成的任务。

### 专题要点

- **社科研究数据类型与AI工具匹配**：问卷数据、访谈文本、网络爬虫数据、政府公开数据、社交媒体数据——每类数据的AI辅助获取路径
- **网页与文档的结构化提取**：无需编程的网页信息抓取；PDF表格与文字的AI提取；非结构化文本→可分析数据集
- **音视频转写与处理**：Whisper等语音识别模型将访谈/演讲非结构化数据转为文本；多语言场景处理
- **AI辅助数据清洗**：利用LLM完成重命名、变量重构、格式统一、缺失值处理；自然语言描述→数据操作
- **合成数据与AI模拟**：多智能体平台（Smallville、AutoGen）作为新型数据产生器的可能性与局限；合成实验的方法论适用场景

### 使用工具

- **网页信息提取**：Firecrawl（自然语言驱动）、ChatGPT + 网页插件
- **文档处理**：Adobe Acrobat AI、Claude（PDF对话与提取）
- **语音转写**：OpenAI Whisper（API或本地部署）
- **AI辅助数据整理**：Claude / ChatGPT（描述操作→生成代码或直接操作）

### 上机练习

- **练习：非结构化文本结构化**：学员选择一份政府报告或访谈文本，使用AI提取关键变量，生成结构化表格
- **案例演示**：音频访谈→文字稿→主题标注的完整链条（10分钟演示）

## M4. LLM驱动的文本分析与测量

**时长**：120分钟
**性质**：讲授 + 上机实践

在政策研究和政治学中，大量核心概念——信任、意识形态、威权主义程度——难以从问卷中直接获得，但可以从文本中测量。本模块从**社科测量的方法论视角**切入LLM文本分析，重点不是"会用工具"，而是"知道在什么条件下能信任输出"。

### 专题要点

- **LLM作为测量工具（LLM as Measurement）**：传统词典法→监督学习→LLM的范式演进；LLM零样本编码的优势（无需标注数据）与局限（可靠性如何验证）
- **情感分析与观点挖掘**：从传统情感词典到预训练模型的演进；LLM情感分析的提示词工程；与人工编码的一致性检验方法（Cohen's Kappa、Krippendorff's α）
- **主题建模与自动分类**：LDA vs. BERTopic vs. GPT分类的比较；开放式问卷答案的AI归类；人-机协作编码流程
- **信息抽取与结构化**：从复杂政策文本中提取可编码的变量；实体识别与关系提取
- **效度与可靠性**：如何验证LLM测量的construct validity；跨模型一致性测试；何时LLM测量**不适合**作为核心变量（方法论红线）

### 使用工具

- **LLM API接口**：OpenAI API / DeepSeek API（无需编程，可通过Colab模板操作）
- **开源NLP工具**：HuggingFace（在线Demo，无需本地配置）
- **可视化与验证**：Google Sheets + GPT插件（轻量数据标注）

### 上机练习

- **练习：LLM编码可靠性验证**：学员使用提供的100条社交媒体评论数据，分别用GPT和人工完成情感标注，计算一致性系数，讨论哪些类型的错误是系统性的
- **案例：开放问卷自动编码**：学员手工总结5条问卷答案的主题，再用ChatGPT完成同样任务，对比两种方法的差异并讨论方法论含义

## M5. AI辅助专业学术写作

**时长**：90分钟
**性质**：讲授 + 工具演示

正式投稿前，研究者常经历一段"写不出来、写不清楚、写得不满意"的煎熬期。本模块围绕学术写作流程中最具体的痛点展开，重点不是"让AI帮你写论文"，而是**让AI成为你写作过程中可控的协作者**。

### 专题要点

- **结果段落的逻辑架构**：如何用AI辅助撰写"结果→解释→对比已有研究"的段落结构；AI提供的初稿与研究者判断之间的边界
- **引用查找与整合**：使用FutureHouse、Semantic Scholar AI辅助查找支撑论据的文献；AI检索结果的核查要求
- **期刊风格迁移**：不同期刊（APSR vs AJPS vs 国内期刊）语言风格的差异；如何构建期刊风格知识库；一键风格迁移的实现与局限
- **图表注释与摘要生成**：利用GPT-4o多模态能力为图表生成规范的描述性文字；从核心论点快速生成摘要初稿
- **发布规范与伦理**：期刊AI使用政策现状（2025年）；什么可以用、什么必须披露；学术诚信的底线

### 使用工具

- **ChatGPT / Claude**：结果段落撰写、讨论启发、语言润色
- **FutureHouse / Elicit**：文献支撑查找
- **Ollama + 本地知识库**（进阶演示）：搭建特定期刊风格微调

### 上机练习

- **练习：期刊风格润色**：学员提交一段自己的结果段落，使用AI工具生成针对目标期刊的风格调整版本，然后小组讨论：哪些修改是纯语言层面的，哪些改变了实质内容
- **案例：图表文字生成**：学员上传一张回归系数图，使用GPT-4o生成图注，评估其准确性与可用性

## M6. 学术传播与综合答疑

**时长**：105分钟
**性质**：工具演示 + 综合实践 + 开放答疑

学术传播不只是发论文，更关乎**如何展示成果、对接合作、扩大影响**。本模块涵盖AI赋能学术传播的核心场景，并通过综合案例将前五个模块的内容串联。

### 专题要点

- **AI辅助Slides与Poster生成**：从论文核心论点快速生成学术报告框架；Beautiful.ai、Gamma等工具的实际使用场景；AI生成 vs 人工设计的质量对比
- **学术邮件与沟通**：联系导师、申请会议、投稿询问的规范邮件生成；多语言学术邮件（中/英）
- **个人学术品牌建设**：个人主页快速搭建（GitHub Pages + AI辅助）；研究项目资料库的构建；学术成果的可复现性报告
- **AI辅助的自动文献追踪**：设置领域关键词的自动监控系统；Research Rabbit、Semantic Scholar Alert的配置

### 综合案例演示（30分钟）

以授课者自身一篇在研论文为案例，**完整演示从文献到写作到传播的AI辅助研究流程**：
1. 用Elicit检索领域文献，构建知识图谱
2. 用Claude辅助假设推演与结果解读
3. 用GPT生成Slides框架与图表注释
4. 展示个人项目主页与可复现报告

### 开放答疑（45分钟）

围绕学员自身研究项目的具体困难，现场示范AI辅助解决路径。
