## 什么是 Vibe Coding for Economists

2025年2月，Andrej Karpathy（前OpenAI研究员）提出"Vibe Coding"：通过自然语言描述与AI对话，完成传统上需要手写代码才能完成的工作。Anton Korinek（弗吉尼亚大学，NBER WP 34202，JEL）将这一范式系统引入经济学研究：

> "Researchers can now build complete analytical tools from simple English descriptions, handling everything from file uploads to regression analysis to visualization — all generated in minutes without writing a single line of code."

**旧模式**：研究者写Stata do-file → 调试语法报错 → 手动生成表格 → 重复十遍

**新模式**：研究者描述分析需求 → AI生成并执行代码 → 研究者审核结果 → 迭代

这不是"让AI替你做研究"，而是**将研究者的时间从语法调试和格式重复中解放出来，集中在真正需要判断力的地方**：研究设计、因果识别策略、结果解读。

Korinek将AI对经济学研究的价值划分为七个维度：文献综述、研究构思、编程与数据分析、数学推导、数据处理、学术写作、研究传播。本工作坊覆盖其中与发表工作流最直接相关的五个。

## M1. 范式转变：从写代码到指导分析

**时长**：90分钟 | **性质**：讲授 + 现场演示

### 问题的起点

在座的研究者，平均每周有多少时间花在Stata/R的语法调试上？花在把回归结果手工整理成表格上？花在因为数据格式不对而反复导入导出上？

这些时间不产生任何研究价值，但它们占据了研究流程的大量精力。Vibe Coding要解决的，正是这个问题。

### AI能力的三次跃迁

Korinek (2025) 将AI能力的演进分为三个范式：

| 范式 | 代表工具 | 对经济学研究者的意义 |
|---|---|---|
| **传统LLM**（2022–2024） | ChatGPT早期版本 | 写邮件、改论文语言、解释概念 |
| **推理模型**（2024–） | GPT-o1, Claude 3.7 | 推导数学、生成计量代码、检查识别策略逻辑 |
| **Agentic系统**（2025–） | Claude Code, OpenAI Codex | 自主执行多步骤任务、全流程研究自动化 |

我们现在处于第三阶段的早期——**会用这套工具的研究者与不会用的之间，生产率差距已经在拉开**。

### 研究者的新角色

**你不是代码的作者，你是分析的导演。**

- 旧角色：理解Stata语法 → 查文档 → 调试 → 得到结果
- 新角色：描述研究需求 → 审核AI的实现方案 → 验证结果 → 迭代

关键技能转变：
- Stata语法知识的重要性**下降**
- 研究设计的清晰表达能力**上升**
- 结果的批判性核查能力**上升**（Trust but Verify）

### 现场演示（30分钟）

**演示1：一句话生成完整回归分析**

口述："我有一个2010-2022年中国A股上市公司面板数据，变量包括ROA、ESG评分、公司规模、负债率、行业代码和年份。帮我跑一个固定效应回归，因变量ROA，核心解释变量ESG，控制变量包括规模和负债率，双向固定效应，按公司层面聚类标准误，输出一个可以直接放论文的系数表格。"

→ Claude Code生成Stata/R代码，解释每一步，运行，输出格式化结果表。

**演示2：识别策略的对话验证**

"我想用DID估计政策X对Y的效应，处理组是受政策影响的城市，控制组是未受影响城市，处理时间2015年。请帮我梳理平行趋势假设的检验方法，并生成对应的预处理检验代码。"

## M2. Vibe Coding数据工程：从原始数据到分析样本

**时长**：105分钟 | **性质**：讲授 + 上机实践

### 数据工作的真实成本

一项典型的管理学/经济学实证研究，数据清洗和整合的时间占比往往超过50%：从CSMAR/Wind导出、变量重命名、多库合并、异常值处理、样本筛选条件……每一步都是纯体力劳动，每一步都可能出错。

AI在这里的价值是**决定性的**：研究者描述需要什么，AI完成具体实现，研究者审核是否符合预期。

### 专题要点

**2.1 描述数据需求，AI写清洗代码**

不需要记住`egen`、`reshape wide`的确切语法。只需要描述：
- "把这个宽格式数据转成长格式，以公司代码和年份为观测单位"
- "将A库的公司代码与B库的统一社会信用代码做模糊匹配，把匹配率低于80%的标记出来人工核查"
- "按行业-年份对ROA做winsorize，上下各1%"

AI生成代码，解释每一步，你负责审核逻辑是否符合研究设计意图。

**2.2 非结构化数据的结构化**

管理世界的读者经常面临的数据问题：
- 上市公司年报：批量提取管理层讨论与分析（MD&A）的关键信息
- 政府工作报告：将政策优先级排序转化为可量化变量
- 企业公告：提取关键事件时间点、受影响主体

演示：将100份上市公司年报上传，用统一提示词模板提取：是否披露ESG报告、员工总数、研发投入、外部审计机构。与人工提取结果做一致性检验。

**2.3 中国特色数据源的AI辅助处理**

- CSMAR/WIND导出的Excel → AI辅助标准化
- 国家统计局、地方统计年鉴 → AI辅助数字提取与对齐
- 工商注册数据、专利数据 → AI辅助企业主体匹配
- 微博/政务微信文本数据 → AI辅助内容编码

**2.4 数据质量的AI辅助核查**

"Trust but Verify"原则——AI做了什么，你必须能看懂并验证：
- 让AI解释每一步清洗操作的逻辑
- 生成数据质量报告（缺失值分布、异常值分布、合并匹配率）
- 关键变量的描述统计与预期不符时，让AI帮你trace back到原始数据

### 使用工具

- **Claude Code**：描述数据操作 → 生成Stata/R/Python代码 → 执行（需要本地配置）
- **Claude.ai + 文件上传**：上传CSV/Excel，直接用自然语言描述清洗需求（零配置，立即可用）
- **Kimi / DeepSeek**：对中文数据描述支持更好，国内访问无障碍
- **ChatGPT Advanced Data Analysis**：上传数据，对话式探索性分析

### 上机练习

**练习（25分钟）：AI辅助数据合并**

提供两个数据集：(1) CSMAR企业财务数据，(2) 专利数据库，需要按公司代码+年份合并，处理不匹配项。学员用AI描述合并逻辑，生成代码，比对结果。

## M3. Vibe Coding计量分析：描述研究设计，AI跑回归

**时长**：120分钟 | **性质**：讲授 + 上机实践

### 核心能力：把研究设计转化为清晰的自然语言描述

这是Vibe Coding在计量分析中最核心的技能。AI能执行分析，但前提是你能把研究设计描述清楚——这本身就是对研究设计质量的一次"语言化测试"。如果你无法用几句话向AI说清楚你的识别策略，你可能也无法向审稿人说清楚。

### 专题要点

**3.1 基准回归的Vibe Coding**

描述格式模板：
```
数据结构：[观测单位] × [时间范围]，[N]个观测
因变量：[Y，含义]
核心解释变量：[X，含义]
控制变量：[列表]
固定效应：[个体FE/时间FE/双向FE]
标准误：[聚类层级]
输出：[期刊格式表格/图形]
```

**3.2 主流因果识别策略的AI辅助实现**

- **双重差分（DiD）**：描述处理组/控制组/处理时间，AI生成平行趋势检验、事件研究图、Callaway & Sant'Anna估计器代码
- **工具变量（IV）**：描述工具变量，AI生成第一阶段F统计量检验、2SLS、弱工具变量robust推断
- **断点回归（RD）**：描述断点和带宽选择逻辑，AI生成McCrary检验、局部线性回归、图形展示
- **合成控制**：描述处理单位和捐献者池，AI生成Abadie-Diamond-Hainmueller估计

**3.3 稳健性检验的系统化**

旧方法：研究者逐一手写每个稳健性检验的代码

新方法：
"基于上述基准回归，系统生成以下稳健性检验：(1)剔除直辖市，(2)剔除2008金融危机年份，(3)替换因变量为ROE，(4)加入行业×年份固定效应，(5)按不同样本期重跑。把所有结果整理成一张稳健性对比表。"

**3.4 Trust but Verify：你必须能读懂AI生成的代码**

- **不能盲信**：AI可能在固定效应设置、样本筛选条件、标准误聚类层级上犯错
- **必须验证**：让AI解释生成代码的每一步；关键系数手工cross-check；样本量是否符合预期
- **方法论红线**：识别策略的合理性判断不能外包给AI；数据生成机制的理解必须是研究者的

### 使用工具

- **Claude Code**（推荐）：在本地项目目录中运行，直接操作数据文件，生成可执行Stata/R代码
- **Cursor + Stata/R**：在Cursor IDE中写do-file，AI辅助补全和调试
- **ChatGPT Advanced Data Analysis**：上传数据，直接运行Python分析，适合快速探索
- **DeepSeek Coder**：国内可用，代码生成能力强，免费

### 上机练习

**练习A（30分钟）：一句话完成基准回归**

学员将自己研究项目的核心回归设定（或使用提供的示例数据）用自然语言描述，AI生成Stata代码，比对：(1) AI的固定效应设置是否与你的意图一致，(2) 聚类标准误层级是否正确，(3) 输出表格是否可以直接用于投稿。

**练习B（30分钟）：平行趋势检验的vibe coding**

描述一个DID设定，让AI生成完整的事件研究图代码，包括置信区间、处理前趋势的统计检验。

## M4. AI辅助学术写作：从结果到投稿

**时长**：105分钟 | **性质**：讲授 + 工具演示

### 重新定义写作中的AI角色

AI不是"枪手"，是"思维提速器"。它最有价值的场景不是生成你不知道该写什么的段落，而是帮你**更快地将已知的思路转化为符合期刊规范的文字**。

对管理世界受众：目标期刊的写作规范、审稿人预期的论证逻辑、结果段落的中文学术语气——这些都可以用AI辅助，前提是研究者对内容的判断是清晰的。

### 专题要点

**4.1 结果段落的结构化写作**

描述你的发现：
- 基准结果：系数方向、显著性、经济意义
- 机制：你相信这个效应通过什么路径产生
- 异质性：哪类子样本效应更强/更弱

AI生成：符合"结果→解读→对比文献→机制讨论"逻辑的段落初稿，研究者的工作是审核实质内容是否准确、删除AI的程式化表达。

**4.2 管理世界 vs 国际期刊的写作差异**

- 管理世界：以中国情境为分析对象，要求说清楚"为什么在中国研究这个问题"；结论部分侧重政策建议；摘要强调贡献而非假设
- 国际顶刊（Management Science, ASQ等）：更强调理论贡献；文献对话的精确性；方法论细节的自我辩护

实操：构建目标期刊的"风格知识库"（上传近5年10篇代表性文章），让AI学习写作风格，生成符合该期刊语气的初稿。

**4.3 审稿意见分诊与回应**

最耗时但最关键的环节。AI的价值：

1. **分诊**：将审稿意见按类型分类（识别策略质疑、数据问题、文献补充、写作修改、概念澄清）
2. **优先级排序**：区分"必须实质性回应"和"可以解释清楚即可"的意见
3. **回应框架**：为每条意见生成"Reviewer's Concern → Our Response → Evidence"结构的初版
4. **补充分析**：描述需要补做的稳健性检验，AI生成代码

**4.4 AI使用的透明度规范（2025年顶刊现状）**

- 管理世界、经济研究：尚无明确AI披露要求，但方法论部分需说明数据处理步骤
- SSCI期刊：多数要求在方法或致谢中说明AI使用；禁止将AI列为作者
- 实践建议：在research notes中记录AI辅助的具体步骤，随时可以说明

### 使用工具

- **Claude.ai**（写作与审稿回应）
- **Kimi**（中文长文档处理，支持大上下文）
- **期刊风格知识库**：Claude Projects + 目标期刊代表文章

### 上机练习

**练习：审稿意见分诊（20分钟）**

学员（或使用脱敏示例）提交一份真实审稿意见，AI执行分诊，生成逐条回应框架。讨论：AI的分类判断与研究者判断在哪里有分歧？

## M5. Research Agent：从Vibe Coding到自主研究流程

**时长**：90分钟 | **性质**：讲授 + 演示

### 从工具到系统

Vibe Coding是入口，Research Agent是终点。Korinek (2025) 展示了经济学研究的自主Agent系统：

- **文献综述Agent**：给定研究主题，自主搜索、筛选、总结文献，生成带引用的综述初稿
- **数据检索Agent**：连接FRED、World Bank等数据库API，自主下载所需指标
- **计量Agent**：给定研究设计，自主生成代码、执行、debug、报告结果
- **多Agent协作**：不同Agent分工（文献、数据、分析、写作），协调完成完整研究流程

### 目前可用的经济学Research Agent工具

**已有开源项目（直接可用）：**
- **Clo-Author**（Hugo Sant'Anna）：17个agents，10个commands，覆盖从文献综述到期刊投稿全流程；`/analyze data.csv`直接跑分析；`/review paper.tex`验证识别策略
- **claude-code-my-workflow**（Pedro Sant'Anna, Emory Econ 730）：为应用计量研究者定制的Claude Code配置

**工具链组合（适合中国情境）：**
- DeepSeek + Cursor：国内可用，代码生成能力强
- Kimi + Python脚本：处理中文数据的Agent工作流
- Claude Code + 本地数据库：适合有一定技术背景的研究者

### 门槛评估：我现在能用什么？

| 技术门槛 | 可用工具 | 适用场景 |
|---|---|---|
| 零门槛（只需浏览器） | Claude.ai, ChatGPT, Kimi | 写作辅助、文献总结、代码解释 |
| 低门槛（会上传文件） | Claude Projects, ChatGPT Advanced Data Analysis | 数据探索、结果解读 |
| 中门槛（会复制粘贴代码） | Cursor, VS Code + Copilot | 代码辅助调试，Stata/R加速 |
| 有门槛（会用命令行） | Claude Code, Clo-Author | 自动化分析流水线 |

**本工作坊结束后，所有参与者至少能使用前两层。**

### 演示：完整Research Agent工作流（30分钟）

以授课者在研项目为案例，演示：
1. 用Elicit自动检索领域文献，提取方法论要点
2. 描述数据需求，AI生成并执行清洗脚本
3. 口述回归设定，AI生成Stata代码并报告结果
4. 将系数表格提交AI，生成结果段落初稿
5. 将审稿意见提交AI，生成回应框架

## M6. 综合实战 + 自带项目工作坊

**时长**：105分钟 | **性质**：个人实战 + 同伴讨论

### 结构

**前45分钟：自带项目上机**

学员将自己当前研究项目的一个具体痛点（数据清洗、回归代码、审稿回应、文献梳理）带到工作坊，在授课者和助教的辅助下，实际操作AI辅助解决。

优先处理类型：
- 数据合并/清洗的技术问题
- 某个计量方法的代码实现
- 正在写的论文段落
- 已收到的审稿意见

**后45分钟：圆桌讨论与答疑**

聚焦三个问题：
1. 哪些环节AI真正帮到了你？哪些环节你发现AI的输出需要高度警惕？
2. 在你的研究领域，AI辅助的方法论创新空间在哪里？（LLM测量、大规模文本分析、合成控制的新数据来源……）
3. 期刊对AI辅助研究的规范正在形成，你准备如何在透明度和效率之间找到平衡？
