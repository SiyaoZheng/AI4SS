# AI赋能社会科学研究工作坊

**主办**：《管理世界》杂志社
**课程类型**：集中工作坊（一天半）
**授课者**：郑思尧（上海交通大学国际与公共事务学院）
**面向受众**：管理学、经济学、公共管理领域科研人员及高年级博士研究生
**定位**：方法论工作坊，非工具培训；以实证研究发表为导向

---

## 核心问题

生成式AI正在改变社会科学研究的**生产函数**——它不只是提高效率，而是在改变哪些研究问题在资源约束下是可行的。

但这一基础设施的设计逻辑源自软件工程，与社科实证研究的方法论要求之间存在**结构性错配**：

| AI系统的优化目标 | 社科实证研究的要求 |
|---|---|
| 生成流畅、令人满意的输出 | 因果识别与内部效度 |
| 基于统计规律的补全 | 构念测量的理论效度 |
| 单次任务完成 | 跨月跨年的研究连续性与可复现性 |
| 黑箱优化 | 分析决策链的可追溯与可解释 |
| 针对英语语料的预训练偏差 | 中国情境数据的特殊性与敏感性 |

本工作坊不是AI工具使用手册，而是帮助研究者**在AI基础设施之上按照社科研究逻辑开展工作**——识别AI输出的系统性偏差，并掌握改造AI行为使之服务于严格实证研究的方法。

## M1. 导论：AI赋能社科研究的机遇、边界与风险

**时长**：90分钟 | **性质**：讲授为主，课堂讨论

生成式AI对社会科学研究的冲击是结构性的：它不是在研究的边缘提供便利，而是在改变研究前沿的可及性边界。曾经需要大型团队才能完成的大规模文本编码、跨语言文献整合、政策文本结构化分析，现在单个研究者即可完成。这是真正的研究力量倍增器——但前提是研究者理解它能做什么，更重要的是，它为何系统性地出错。

### 专题要点

- **LLM的工作机制（研究者版）**：自注意力机制的直觉理解；Token预测与统计推断的本质差异；为什么语言模型能给出"合理"但错误的输出；训练数据偏差如何系统性地影响中国议题的分析质量
- **能力边界的三个维度**：
  - 能做：零样本编码、跨语言语义整合、大规模文本结构化、假设发散与文献脉络梳理
  - 慎用：作为主要测量工具（需要效度验证）、因果推断步骤、数据生成
  - 不能替代：理论建构、研究设计判断、内部效度评估
- **主流模型的中国情境适用性**：GPT-4o、Claude 4、DeepSeek、Kimi、通义千问在中文学术文本处理上的差异；访问限制与数据隐私的制度性考量
- **方法论伦理的新问题**：AI辅助研究的披露规范（顶刊现状）；数据隐私与AI服务提供商；中国政策敏感数据的本地化处理原则

### 工具演示

- **模型横向比较**：同一段政策文本，分别提交DeepSeek、Claude、GPT，比较其提取结果的差异性与系统性偏差
- **幻觉压力测试**：要求模型引用《管理世界》《经济研究》上的具体文献，实时在知网验证

### 课堂讨论

**三分钟写下**：你当前研究项目中，哪个环节最耗时且技术含量最低？我们用这些真实案例作为后续模块的演示素材。

## M2. AI辅助文献调研与研究设计

**时长**：105分钟 | **性质**：讲授 + 上机实践

文献调研的核心难点不是找到文献，而是**理解文献之间的演化逻辑、定位自身研究的边际贡献**。审稿人拒稿最常见的理由之一，是综述部分无法清晰说明"这篇文章相对于X解决了什么新问题"。AI在这里的价值，不是替代阅读，而是帮助研究者在有限时间内建立更完整的领域地图。

### 专题要点

- **从关键词检索到语义搜索**：传统数据库检索（CNKI、Web of Science）的局限；AI驱动的语义相关性如何发现被关键词遗漏的核心文献；中英文文献的跨语言整合策略
- **文献脉络的结构化分析**：如何用AI快速识别领域内的"理论节点文献"与"方法论转折点"；构建文献演化图谱（理论分歧在哪里、方法论争议在哪里、经验发现是否一致）
- **研究空白的精准定位**：AI辅助识别"已研究但结论不一致的问题"与"结构性未研究的问题"；区分真正的研究空白与表述性空白
- **假设生成的AI辅助流程**：LLM作为理论推演的思想实验工具；如何用AI发散假设、再用研究者判断收敛；AI建议的假设为何往往缺乏理论机制——以及如何补救
- **中国情境文献的特殊处理**：政策文件作为文献来源；中文学术语境与英文概念的对应与错位；避免"中国案例套西方理论"的文献处理陷阱

### 使用工具

- **语义文献搜索**：Elicit、Semantic Scholar AI摘要、Perplexity学术模式
- **文献知识图谱**：Connected Papers、Research Rabbit（中文文献局限说明）
- **长文档对话**：Claude（上传PDF批量提问）、Kimi（对中文文献支持更好）
- **CNKI整合**：知网AI摘要功能 + 手动导出结合Claude分析

### 上机练习

**练习A（25分钟）：文献脉络快速梳理**
学员选择自己近期阅读过的5–8篇核心文献，将摘要或全文提交给Claude，要求其：(1) 识别理论分歧点，(2) 梳理方法论演进，(3) 指出经验发现不一致之处。然后与自己已有的理解对比：AI漏掉了什么？误读了什么？

**练习B（20分钟）：研究边际贡献的语言精确化**
学员将自己一篇在写或在投论文的"研究贡献"部分提交，要求AI模拟审稿人视角提出质疑："相比X文章，这篇的贡献是什么？"比较AI质疑与真实审稿意见的重叠度。

## M3. AI辅助数据获取与预处理：中国情境的特殊挑战

**时长**：105分钟 | **性质**：讲授 + 工具演示

管理学和经济学实证研究的数据工作，在中国情境下有其特殊性：上市公司年报、政府工作报告、政策文本、企业公告、统计年鉴——这些数据大量存在，但格式杂乱、来源分散、标准不统一。AI在数据预处理环节能显著减少这类体力劳动，将研究者的精力集中在真正需要判断的地方。

### 专题要点

- **非结构化文本的结构化提取**：上市公司年报关键信息提取（风险披露、管理层讨论与分析、关联交易）；政策文本的维度编码；新闻报道的事件信息抽取
- **多源数据的AI辅助清洗与整合**：多个数据库（CSMAR、Wind、CNRDS）导出数据的格式标准化；变量命名规范化；合并键的自动识别与错误检测
- **大批量文档处理流程**：批量PDF提取的工程化方案（不需要写代码：Claude Projects + 系统提示词 vs API Colab模板）；音视频资料的转写（政府新闻发布会、企业业绩说明会）
- **数据质量的AI辅助验证**：利用LLM进行数据异常值的语义核查；AI辅助的抽样验证流程
- **合成数据与Agent模拟的方法论地位**：作为理论预测的模拟检验 vs 作为替代经验数据——边界在哪里；顶刊对合成数据的接受度现状

### 使用工具

- **文档批量处理**：Claude Projects（上传多份PDF，统一提取）；Kimi（对中文文档处理能力强）
- **语音转写**：通义听悟（对中文普通话识别率高）；Whisper（方言/专业术语场景）
- **数据清洗**：Claude / ChatGPT（描述清洗逻辑→生成Python/R代码→执行）；无需手写代码
- **年报/公告抓取**：巨潮资讯AI辅助 + Claude提取结构化信息

### 演示案例

**案例：100份上市公司年报的ESG信息提取**
演示如何使用Claude Projects，通过统一提示词模板从年报"社会责任"章节提取：员工人数、薪酬披露质量、环保支出、公益捐赠。对比人工提取与AI提取的一致性，讨论哪类信息需要人工复核。

## M4. LLM作为测量工具：文本分析的方法论边界

**时长**：120分钟 | **性质**：讲授 + 上机实践

这是全程方法论含量最高的模块。在管理学和经济学中，越来越多研究使用LLM对政策文本、企业公告、新闻报道进行编码——将其作为构建新变量的手段。这一做法具有真正的方法论创新价值，但也面临严格的效度挑战。**顶刊审稿人已经开始系统性地质疑LLM测量的construct validity。** 本模块帮助研究者在这一前沿方法上做到既敢用又能防守。

### 专题要点

- **LLM测量的范式定位**：传统词典法 → 监督学习 → 零样本LLM的方法论演进；LLM测量的核心优势（无需标注数据、语义理解、跨语言）与核心风险（提示词依赖、模型偏见、不可复现）
- **政策文本分析的实证应用**：政策意图强度的LLM编码（以环保政策为例）；政府工作报告的议题优先级提取；公文措辞的情感强度测量
- **企业公告与财报的文本分析**：管理层语调（managerial tone）测量；业绩说明会的不确定性词汇；年报风险披露的内容变化
- **效度验证的完整流程**（这是审稿人要问的）：
  - 内容效度：提示词设计是否准确操作化了构念
  - 收敛效度：与已有量表/人工编码结果的相关性（Cohen's Kappa、Spearman ρ）
  - 区分效度：是否能区分应当不同的构念
  - 跨模型稳健性：换一个LLM结果是否稳定
  - 随机化提示词测试：结果是否对提示词细微变化过度敏感
- **方法论红线**：哪些情况下LLM测量目前不宜作为论文核心变量；如何在论文方法论部分进行恰当的局限性说明

### 使用工具

- **LLM批量编码**：DeepSeek API + Colab模板（提供即用模板）；Claude Batch API
- **效度验证计算**：Python（提供脚本）或SPSS（传统方法对照）
- **中文情感分析基准**：SnowNLP、BERT-CCPoem等中文预训练模型作为对照基线

### 上机练习

**练习：LLM编码的效度验证**
学员使用提供的200条企业年报风险披露句子（已有人工"高/中/低风险"标注），用DeepSeek零样本编码，然后计算与人工标注的一致性系数。分组讨论：
- 哪类句子LLM系统性地与人工标注不一致？
- 提示词如何修改可以改善一致性？
- 如果这是论文中的核心变量，需要补充什么效度证据？

## M5. AI辅助写作：从结果呈现到投稿策略

**时长**：90分钟 | **性质**：讲授 + 工具演示

对于已有稳健发现的研究，写作阶段的核心挑战不是"写什么"，而是**如何以符合目标期刊逻辑的方式呈现**。管理世界与经济研究的写作规范不同，国内顶刊与国际顶刊的叙事结构差异更大。AI在这里最有价值的作用，是帮助研究者快速完成风格迁移，以及在结果解读环节提供"挑战性"的对话。

### 专题要点

- **结果段落的逻辑架构与AI辅助**：主效应→机制检验→异质性分析的叙事逻辑；如何用AI辅助生成"结果→解读→对比已有研究"的完整段落框架；AI提供的初稿与研究者内容判断的分工
- **期刊风格迁移的实操**：管理世界 vs 经济研究 vs 社会学研究 vs SSCI期刊的写作风格差异；构建目标期刊本地知识库（上传近10篇文章，提取风格规律）；一键风格润色的操作与局限
- **图表文字的高效生成**：将回归系数图/机制路径图提交GPT-4o，生成规范的描述性图注；避免图注中引入原始数据中没有的解读
- **摘要与关键词的优化**：AI辅助根据目标期刊读者结构优化摘要；高引关键词的策略性选择
- **回应审稿意见的AI辅助**：将审稿意见提交Claude，要求其(1)分类意见类型，(2)识别核心关切，(3)草拟回应逻辑框架；研究者在此基础上完成实质性回应

### 使用工具

- **写作润色**：Claude（逻辑结构和语言风格）；Kimi（中文学术语气）
- **期刊风格知识库**：Claude Projects（上传目标期刊代表文章作为参考语料）
- **图表解读**：GPT-4o / Claude（多模态图表上传）

### 上机练习

**练习：审稿意见分诊**
学员（或授课者提供脱敏案例）将一份真实审稿意见提交Claude，要求其：(1) 按关切类型分类（理论、方法、数据、文献、表达），(2) 识别哪些是必须实质性修改的"硬性意见"，(3) 生成逐条回应的框架。小组讨论：AI的分类与研究者的判断在哪里有分歧？

## M6. 顶刊审稿逻辑与AI辅助修稿 + 综合答疑

**时长**：105分钟 | **性质**：讲授 + 综合演示 + 开放答疑

### 专题讲授（40分钟）

**顶刊审稿逻辑与AI辅助的战略定位**

管理学和经济学顶刊的审稿标准正在发生微妙变化：一方面，AI辅助的研究能够在数据规模和文本处理上超越过去；另一方面，审稿人对AI生成内容持日益警惕的态度，方法论上的"AI捷径"正在成为被质疑的靶点。

- **审稿人视角的AI使用信号识别**：哪些写作模式会暴露过度依赖AI（过于流畅的综述、缺乏具体细节的方法论说明、格式化的贡献表述）
- **AI使用的透明度策略**：何时主动披露、如何披露、如何在方法论部分恰当说明LLM测量；不同期刊的AI披露政策现状（管理世界、经济研究、JFE、ASQ等）
- **AI赋能研究的可复现性标准**：提示词的版本记录；使用模型的参数说明；数据可用性声明的新挑战

### 综合案例演示（25分钟）

以授课者自身一篇在研/在投论文为案例，**完整展示AI辅助的研究流程节点**：

1. 文献脉络梳理：如何用Elicit识别领域内方法论转折点
2. 政策文本编码：DeepSeek批量提取变量的提示词设计与效度验证步骤
3. 结果解读：Claude辅助生成机制讨论段落的初稿
4. 审稿意见回应：真实R&R过程中AI辅助分诊的使用场景

### 开放答疑（40分钟）

围绕学员当前研究项目的具体困难，现场示范AI辅助解决路径。优先处理：
- 正在处理的数据难题
- 正在写作或修改的稿件
- 审稿意见的应对策略
